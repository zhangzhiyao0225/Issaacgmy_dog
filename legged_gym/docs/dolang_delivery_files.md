# Dolang ONNX 与配置文件交付清单

本文档用于限定本项目中我方交付物范围。交付内容仅覆盖策略 ONNX 文件、与其匹配的配置文件及必要的文件说明，不包含机器人硬件、实机部署程序、驱动、中间件、现场调试或第三方系统集成。

## 1. 交付范围

我方负责交付以下文件：

| 序号 | 文件/目录 | 说明 | 必交付 |
| --- | --- | --- | --- |
| 1 | `policy.onnx` | 训练完成后导出的策略网络 ONNX 文件，用于合作方后续部署或推理集成 | 是 |
| 2 | `dl_config.py` | 与 ONNX 对应的 dolang 机器人配置文件，包含观测维度、动作维度、默认关节角、控制参数、奖励参数、资产路径等 | 是 |
| 3 | `onnx_export_info.md` | ONNX 导出说明，记录导出日期、模型版本、输入输出维度、观测顺序、动作顺序、配置文件版本、校验摘要 | 是 |
| 4 | `checksums.txt` | 交付文件的 SHA256 或 MD5 摘要，用于确认文件未被修改 | 是 |
| 5 | `README_delivery.md` | 交付包简要说明，列出文件用途、对应关系、已知前提和不包含项 | 建议 |

如需在本仓库内直接运行 `--task dolang`，可额外提供任务注册相关文件，例如：

| 文件 | 说明 |
| --- | --- |
| `legged_gym/legged_gym/envs/dolang/dl_config.py` | dolang 主配置文件 |
| `legged_gym/legged_gym/envs/__init__.py` | 任务注册文件，用于注册 `dolang` 任务 |

上述仓库内辅助文件仅用于复现或定位配置，不代表我方负责合作方部署框架的接入实现。

## 2. 建议交付包结构

```text
dolang_onnx_delivery_YYYYMMDD/
  model/
    policy.onnx
  config/
    dl_config.py
  docs/
    onnx_export_info.md
    README_delivery.md
    checksums.txt
```

如存在多个版本，建议使用明确版本号：

```text
policy_dolang_v001.onnx
dl_config_dolang_v001.py
onnx_export_info_v001.md
```

## 3. 配置文件应包含的关键信息

`dl_config.py` 至少应明确以下内容：

| 项目 | 当前 dolang 配置参考 |
| --- | --- |
| 动作维度 | `num_actions = 12` |
| 单步观测维度 | `num_one_step_observations = 45` |
| 总观测维度 | `num_observations = 270` |
| 默认站立高度 | 约 `0.47 m` |
| 默认站立关节角 | `hip = 0.0`, `thigh = 0.75`, `calf = -1.5` |
| 机器人 URDF 路径 | `resources/robots/dolang/urdf/dolang.urdf` |
| 足端名称匹配 | `foot_name = "foot"` |
| 动作缩放 | `action_scale = 0.3` |
| 控制类型 | `control_type = "P"` |

如合作方部署时不使用本仓库配置文件，应由合作方根据 `dl_config.py` 自行映射其部署配置。

## 4. ONNX 导出说明应包含的内容

`onnx_export_info.md` 建议记录：

| 项目 | 内容 |
| --- | --- |
| 模型名称 | 例如 `dolang_flat_policy_v001` |
| 导出日期 | `YYYY-MM-DD` |
| 训练配置 | 对应的 `dl_config.py` 文件名和摘要 |
| 输入张量 | 张量名称、shape、dtype |
| 输出张量 | 张量名称、shape、dtype |
| 观测约定 | 观测项顺序、历史帧数量、归一化/裁剪约定 |
| 动作约定 | 12 个关节顺序、动作缩放、默认关节角 |
| 运行前提 | 合作方需保证部署端观测、动作、单位和坐标系与配置一致 |
| 文件摘要 | ONNX 与配置文件校验值 |

## 5. 验收标准

交付验收仅针对文件完整性和接口一致性：

1. `policy.onnx` 文件存在，且可被标准 ONNX 工具加载。
2. `dl_config.py` 文件存在，且记录的动作维度、观测维度、默认关节角与 ONNX 导出说明一致。
3. `checksums.txt` 中的校验摘要与交付文件一致。
4. 合作方确认已收到 ONNX 文件、配置文件和导出说明。
5. 如需我方协助解释接口，仅限 ONNX 输入输出、配置字段含义、观测动作顺序和导出版本对应关系。

## 6. 不包含项

以下内容不属于本次交付范围，除非双方另行书面确认：

1. 实机部署代码、推理服务、ROS/ROS2 节点、SDK 接入和通信协议适配。
2. 电机驱动、低层控制器、状态估计器、传感器驱动、遥控器或急停系统。
3. 机器人硬件设计、URDF/CAD 正确性审核、动力学参数标定和电机参数标定。
4. 实机安全验证、场地测试、跌倒保护、限位保护和安全责任。
5. 合作方平台中的模型加载、TensorRT/OpenVINO/其他推理引擎转换与优化。
6. 因合作方修改观测、动作、单位、关节顺序、控制频率或坐标系导致的部署问题。
7. 超出已确认配置版本的模型重训、重新导出或新增功能。

## 7. 版本与变更

每次正式交付应固定以下信息：

| 项目 | 记录方式 |
| --- | --- |
| ONNX 文件版本 | 文件名或版本号 |
| 配置文件版本 | 文件名、提交号或摘要 |
| 机器人数据版本 | URDF 文件名、机械参数确认日期 |
| 交付日期 | `YYYY-MM-DD` |
| 变更原因 | 初次交付、参数更新、重新训练、重新导出等 |

合作方若更改 URDF、关节顺序、控制周期、传感器输入、部署控制器或硬件参数，应视为新输入条件，是否重新训练或重新导出由双方另行确认。
